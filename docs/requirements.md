# MIDI FUNFUN 要件定義書 v2

> v1 からの主な変更点は末尾「付録A: v1 からの変更履歴」に記載。

## 1. 概要

自分の声(ボーカル・鼻歌)を録音し、ピッチ解析によってMIDIノートに変換、ピアノロール上で編集・スケール補正した上で、DAWやプラグインへMIDIファイルとして書き出すためのツール。

- **目的**: 自分の楽曲制作で使う実用ツール
- **想定ユーザー**: 開発者本人(1人)
- **開発方針**: JUCE (C++) を用い、実装はClaude Codeにほぼ全面的に委任する
- **名称**: MIDI FUNFUN

## 2. ターゲット環境

| 項目 | 内容 |
|---|---|
| フレームワーク | JUCE (C++) |
| ビルド形式 | スタンドアロンアプリ(メイン)、VST3プラグイン(サブ、同一コードベースから生成) |
| OS | Windows 11(最優先)。将来的にmacOS対応を見据え、Windows依存APIは極力避けた実装にする |
| 想定ホスト(参考) | Fender Studio Pro(PreSonus Studio Oneベース)。ただしスタンドアロン利用がメインのため、ホスト依存機能は前提としない |
| MIDI書き出し先の例 | Toontrack EZbassのMIDI Drop Zoneなど、外部ファイルのドラッグ&ドロップを受け付けるプラグイン/DAW |

## 3. 全体ワークフロー

1. アプリを起動(スタンドアロン)、またはDAWにVST3としてロード
2. 入力デバイス(マイク)を設定画面で選択
3. BPMを手動入力(クオンタイズの基準として使用)。必要に応じてメトロノーム/カウントインをON
4. 「録音」ボタンで録音開始(録音中はモニタリング音を再生。入力レベルメーターで音量を確認)
5. 「解析」ボタンを手動で押下し、録音済み音声からピッチを検出、MIDIノート化(半音単位)
6. ピアノロールUI上でノートを確認・編集(背後に波形と連続ピッチカーブを表示)
7. 必要に応じて「クオンタイズ」を適用(強度を指定可能)
8. 必要に応じて選択ノートに「スケール補正」を適用
9. 内蔵ピアノ音でプレビュー再生し、仕上がりを確認
10. MIDIをドラッグ&ドロップ、または名前を付けて保存する形で書き出す

## 4. 機能要件

### 4.1 録音機能

- アプリ内でマイクからの音声録音が可能(JUCEのAudioDeviceManagerを利用)
- 入力デバイス・サンプルレート・バッファサイズを選択できる設定画面を用意
- **入力はモノラルを前提とする**(マイク1本。ステレオ入力が来た場合は片チャンネル、またはミックスダウンして1チャンネルとして扱う)
- **入力レベルメーターを表示する**。録音中の入力音量をリアルタイムで確認でき、小さすぎる/クリップしている状態が視覚的にわかるようにする(ピークホールド表示があると望ましい)
- 録音中は自分の声をモニタリングできる(ヘッドホン使用を想定)
- 手動入力したBPMに基づくメトロノーム/カウントインを再生できる
- 1回のアプリ起動セッション内であれば、複数回の録音(テイク)を行い、比較・選択できる
- **1テイクの最大録音長は5分とする**。5分に達したら自動的に録音を停止する
- **録音バッファのメモリ上限を設ける**。
  - 目安: モノラル・48kHz・32bit floatで約 192KB/秒 → 5分テイクで約55MB
  - 全テイク合算の上限を **512MB(5分テイク約9本相当)** とし、これを超える新規録音を試みた場合は警告を表示して録音を開始しない(または古いテイクの削除を促す)
- アプリ再起動をまたいだ録音データ・解析結果・編集内容の保存/復元は不要(MVPでは対象外)

### 4.2 ピッチ検出・MIDI変換

- 入力: モノフォニック(単音)のボーカル・鼻歌を前提とする(和音・複数音源には非対応)
- 出力: 半音単位に量子化されたMIDIノート(ピッチベンドは使用しない)
- **ピッチ検出アルゴリズム(確定)**: 外部ライブラリ(aubio等)は使用せず、**YIN系アルゴリズムを自前実装する**(純粋なC++/DSPコードとしてJUCEプロジェクトに同梱し、外部依存を持たない)。
  - **第1段階**: 素のYINを実装する。単音ボーカル/鼻歌かつ入力レベルが適切であれば、まずこれで実用精度を目指す
  - **第2段階(必要に応じて)**: ピッチのふらつきが実用上問題になった場合に限り、pYIN(確率的平滑化/Viterbi)相当の後処理を追加する
  - 解析はオフライン(「解析」ボタン押下時のバッチ処理)であり、リアルタイム性能の制約は考慮しなくてよい。精度・安定性を優先する
  - ライブラリ非依存としたのは、クロスプラットフォームのビルドを単純に保ち(Windows優先・macOS移植を妨げない)、ライセンス上の制約を持ち込まないため
- 解析は自動実行ではなく、「解析」ボタンを押したタイミングで手動実行する
- ノイズゲート/無音検出により、ブレスノイズや無音区間を誤ってノート化しないようにする(感度はUIから調整可能)
- 極端に短いノート(ピッチのチラつきによる誤検出)を除去する最小ノート長フィルタを設ける
- **オクターブエラー対策を行う**。鼻歌・ボーカルで頻発する「単発でオクターブ上/下に飛ぶ」誤検出を抑制する。前後のノートから極端に外れた孤立したオクターブジャンプを検出し、周辺の音高に合わせて補正する(しきい値は内部固定でも可)
- 感度・しきい値をUI上のスライダーで調整できる

### 4.3 ピアノロール編集

- 検出されたMIDIノートをピアノロール形式で表示・編集する
- 編集操作:
  - ノートの音高移動(縦方向ドラッグ)
  - ノートのタイミング移動・長さのリサイズ(横方向ドラッグ)
  - ノートの削除
  - ノートの新規手動追加
- 選択操作:
  - 単一クリックで1つ選択
  - Shift/Ctrlクリックで複数選択
  - ドラッグによる矩形(マーキー)選択
  - 全選択機能
- **キーボードショートカット**を用意する:
  - 選択ノートの削除: `Delete`
  - Undo: `Ctrl+Z`
  - Redo: `Ctrl+Y`
  - 全選択: `Ctrl+A`
- **グリッドスナップの切り替え(Gridボタン)**:
  - ツールバーに「Grid」トグルボタンを設ける
  - Grid が **ON** のとき: ノートのドラッグ移動・リサイズ・新規追加はグリッドにスナップする
  - Grid が **OFF** のとき: スナップせず自由に移動・リサイズできる
  - ※この Grid ON/OFF は「編集ドラッグ時のスナップ挙動」を制御するもので、4.4の「クオンタイズ」ボタン(既存ノートを一括でグリッドに揃える処理)とは別機能とする
- 編集操作(クオンタイズ・スケール補正含む)すべてに対してUndo/Redoに対応する
- 元の録音波形を、ピアノロールの背後に半透明で重ねて表示する(Melodyneのような見た目)
- **連続ピッチカーブを表示する**。半音に量子化する前の生の音程曲線(ゆらぎを含む)を波形の上に薄く重ねて表示し、「検出がどこを拾っているか」を視覚的に確認できるようにする
- 声域(おおよそ2オクターブ程度)に応じて縦方向の表示範囲を自動フィットする。手動ズームも可能とする
- 再生ヘッド(現在位置)を表示する
- ノートをクリックすると、その音高を単音プレビュー再生できる

### 4.4 クオンタイズ

- 手動入力されたBPMを基準に、拍グリッドへのスナップを行う
- 「クオンタイズ」ボタン押下時のデフォルト動作: ノートの開始位置のみをグリッドに揃える
- **クオンタイズ強度(0〜100%)を指定できる**:
  - 100% = 完全にグリッドへ吸着
  - 中間値 = 元の位置とグリッド位置の間に比例して寄せる(歌の走り・もたりを残しつつ整えられる)
  - UI上でスライダーまたは数値入力により指定する
- ボタンを右クリックすると、以下を選択できるコンテキストメニューを表示する:
  - 開始位置のみをグリッドに揃える(デフォルト)
  - 開始位置と長さの両方をグリッドに揃える

### 4.5 スケール補正

- 選択中のノートに対してのみ適用する(全ノート一括ではなく、対象を選べる)
- ルート音とスケールの種類を指定できる
  - メジャー、マイナー
  - 一般的なモード(ドリアン、ミクソリディアンなど)
  - ペンタトニック、ブルース
- スケール外の音は、上下どちらか近い方の最近接スケール音に補正する
- 補正強度は常に100%(段階調整は行わない、MVPではシンプルに固定)

### 4.6 プレビュー再生

- シンプルな内蔵ピアノ風の音色でMIDIノートを再生する(音高確認が目的のため、リアルな音質は求めない)
- 個別ノートクリックによる単音プレビューにも対応する

### 4.7 ベロシティ設定

- 設定項目として「ベロシティのデフォルト値」を用意し、数値入力(直接入力)で変更できるようにする
- デフォルト値: 90
- 解析によって生成されるMIDIノート、および手動追加するノートには、この設定値がベロシティとして適用される

### 4.8 MIDI書き出し

- **MIDIファイル仕様**:
  - フォーマット: 標準MIDIファイル(SMF)
  - **時間分解能(PPQ / TPQN): 480 を推奨値とする**(多くのDAWで扱いやすい標準的な解像度)
  - **MIDIチャンネル: 全ノートをチャンネル1に固定**(単音メロディ1本の書き出しのため、チャンネルの使い分けは行わない)
  - テンポ情報として手動入力したBPMをファイルに含める
- 主要な書き出し方法: OSレベルの外部ファイルドラッグ&ドロップ(JUCEの`performExternalDragDropOfFiles`相当)により、生成した一時MIDIファイルをDAWや他プラグイン(例: Toontrack EZbassのMIDI Drop Zoneなど)へドラッグして渡せるようにする
- 補助的な書き出し方法: 標準的な「名前を付けて保存」ダイアログでMIDIファイルをディスクに保存できるようにする(ドラッグ&ドロップが環境によって機能しない場合の保険)
- **一時ファイルの管理**: ドラッグ&ドロップ用に生成する一時MIDIファイルは、置き場所(OSの一時ディレクトリ)と削除タイミングを定める。アプリ終了時、または一定数を超えた場合に古い一時ファイルを削除し、ゴミが溜まらないようにする

## 5. UI/デザイン要件

- 配色イメージ: 「青空」を意識した水色系のカラーパレット
- 参考にしたい雰囲気: Melodyneのような、波形とノートを重ねて見せるスタイル
- 画面構成(イメージ):
  - 上部ツールバー: 録音・解析ボタン、入力レベルメーター、BPM入力、Gridトグルボタン、クオンタイズボタン(+強度指定 +右クリックメニュー)、感度スライダー、スケール補正設定(ルート/スケール種類選択+適用ボタン)、MIDI書き出し(ドラッグアイコン+保存ボタン)
  - 中央: 左側に鍵盤、右側にピアノロール(波形オーバーレイ、連続ピッチカーブ、ノートブロック、再生ヘッド)
  - 下部: 再生トランスポート、プレビュー音源表示

## 6. 非機能要件

- 実装言語/フレームワーク: C++ / JUCE
- 開発者(ユーザー)はC++/JUCEの実装経験がほぼないため、Claude Codeが実装の大部分を担当する前提で、コードには可読性の高いコメントや構成を心がける
- Windows 11での動作を最優先とし、将来的なmacOS移植を妨げないコード設計とする

### 6.1 VST3プラグインとしての最低要件

- **プラグイン状態の保存(getStateInformation / setStateInformation)**: 編集内容の永続化はMVPでは不要だが、VST3としてホスト(Studio One等)にロードされる以上、ホストは保存時に必ずプラグイン状態の取得を試みる。**状態を保存しない場合でも、クラッシュしない安全な空実装(またはバージョン識別子のみを返す最小実装)を必ず用意する**
- **VST3でのドラッグ&ドロップ**: `performExternalDragDropOfFiles`はスタンドアロンでは素直に動作するが、VST3としてホスト内で動作する場合はホストのウィンドウ管理やスレッド制約で機能しないことがある。**VST3ではドラッグ&ドロップは動作保証対象外とし、「名前を付けて保存」を主たる書き出し手段とする**
- **マイク録音とVST3の関係**: VST3プラグインはホストのオーディオI/Oに乗るのが基本であり、プラグイン自身が独立してマイク入力を掴む構成はホストと競合しやすい。**録音機能はスタンドアロン版を主対象とし、VST3版での録音は動作保証対象外**とする

## 7. MVPスコープ外(将来検討事項)

- DAWプロジェクトの保存・再読み込みをまたいだ解析結果・編集内容の永続化
- 和音・複数音源(ポリフォニック)入力への対応
- スケール補正の強度調整(0-100%の段階指定)
- ピッチ検出の信頼度(曖昧な区間)の視覚化
- MIDIノートのベロシティを入力音量に連動させる機能(MVPでは設定画面のデフォルト値による固定ベロシティとする)
- **キー/スケールの自動推定**: 録音した歌唱で使われている音を集計し、推定されるキー/スケールをスケール補正設定として提案する機能

## 8. 未決定事項

現時点で残っている未決定事項はなし。
(v2で残していた「ピッチ検出の実装手段」は、YIN系の自前実装で確定。詳細は4.2を参照)

## 付録A: v1 からの変更履歴

本v2で追加・変更した項目は以下のとおり。

- **4.1 録音**: 入力をモノラル前提に明記 / 入力レベルメーターを追加 / 最大録音長5分を追加 / メモリ上限(合計512MB目安)を追加
- **4.2 ピッチ検出**: 検出アルゴリズム(pYIN第一候補)を明記 / オクターブエラー対策を追加
- **4.3 ピアノロール**: キーボードショートカット(Delete / Ctrl+Z / Ctrl+Y / Ctrl+A)を追加 / GridボタンによるスナップON/OFF切り替えを追加 / 連続ピッチカーブ表示を追加
- **4.4 クオンタイズ**: 強度指定(0〜100%)を追加
- **4.8 MIDI書き出し**: MIDIファイル仕様(PPQ 480推奨 / チャンネル1固定 / BPM埋め込み)を明記 / 一時ファイルのクリーンアップを追加
- **5 UI**: 上記追加要素(レベルメーター、Gridボタン、クオンタイズ強度、連続ピッチカーブ)をツールバー/画面構成に反映
- **6.1(新設) VST3最低要件**: 状態保存の空実装 / VST3でのD&D非保証 / VST3録音非保証を明記
- **7 スコープ外**: キー/スケール自動推定を追加
- **8 未決定事項**: ピッチ検出の実装手段を「YIN系の自前実装(段階導入・ライブラリ非依存)」で確定し、未決定事項を解消
